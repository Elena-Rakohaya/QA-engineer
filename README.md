## Проекты

Проект 1: тест новой функциональности веб-приложения для учителей от Skyeng -работа с личными событиями в Календаре.

Что нужно было сделать:

<p>Задача №1. Тестирование требований.<br>
Задача №2. Функциональное тестирование.<br>
Задача №3. Приемочное тестирование.<br>
Задача №4. Smoke тестирование.<br>
Задача №5. Регрессионное тестирование.<br>
Задача №6. API.</p>

<p>Для выполнения поставленных задач была разработана тестовая документация: чек-лист и тест-кейсы. Для создания тестовой документации были использованы следующие инструменты: sitechco, qase.io, Postman. В ходе тестирования были выялены баги. Для фиксации обнаруженных ошибок был использован инструмент Jira.<br>
[Ссыллка на проект](https://drive.google.com/drive/folders/1ehm-K_w2L5ODd515b8frJ02sefOzVzZC?usp=sharing) .</p>

Выводы: 
1. Новый функционал (личные события) системы готов к выпуску. 
2. Новый функционал работает, но имеет два дефекта, которые можно исправить в короткий срок.
3. выпускать весь продукт на пользователей (портал) нельзя. Так как прохождение чек-листа регресс тестирования показал 2 критических бага, которые необходимо исправить в кротчайший срок.
